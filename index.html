Digital Publishing Tools for WordPress

Digital Publishing Tools for WordPress is a plugin that allows anyone to create HTML articles for Digital Publishing Solution directly from WordPress.

Download Now
Prerequisite Knowledge

Experience with WordPress (Content Management System) and knowledge of the Adobe Digital Publishing Solution is required. This plugin also assumes you have access to a web server and have basic HTML/CSS knowledge.

Requirements

WordPress 3.5 +: PHP 5.2.4 or higher, MySQL 5.0 or higher, Apache or nginx recommended. FTP access to the server to install the plugin.

Optional but recommended

Adobe Digital Publishing Solution API access (see how to get access to the Adobe Digital Publishing Solution API)

Adobe Digital Publishing Solution API

Access to the Adobe Digital Publishing Solution API is recommended for this plugin. Without the API access the plugin will still allow you to create and manage articles, edit, and download articles in an Adobe Digital Publishing Solution .article format. Access to the Adobe Digital Publishing Solution API will unlock cloud publishing functions in the plugin. This will allow you to directly publish articles from WordPress directly to Adobe Digital Publishing Solution. API access will also streamline the process of managing and creating articles in Adobe Digital Publishing Solution.


How to gain access to the API

API access is available for enterprise customers of Adobe Digital Publishing Solution. You only need one API key for the plugin. To request API access, have the account manager of the Adobe Digital Publishing Solution enterprise account e-mail: wwds@adobe.com.


Settings page
You'll learn how to get the information for the Adobe API fields in the below steps.


Setting up the plugin with API access

  1. Set up an Adobe ID
  2. Whitelist the Adobe ID
  3. Create Device Token using the Adobe ID
  4. Enter the API information into the plugin

1. Set up an Adobe ID

Once you have an API key you will want to create an AdobeID for access to the API. For security, do not associate your Master Admin AdobeID with the API Key. We recommend creating a new AdobeID (https://accounts.adobe.com/). Then use the Adobe Digital Publishing Solution portal to assign that AdobeID the roles and permissions it needs for the project you’re using for the plugin. You can assign roles using the Adobe Digital Publishing Solution dashboard: (https://publish.adobe.com/admin/index.html#/users).

___

2. Whitelist the Adobe ID

After you have an AdobeID that has been assigned to the permissions/roles of the project you’d like to use with the plugin, you will need to e-mail: wwds@adobe.com asking for that account to be whitelisted.

___

3. Create a Device Token using the Adobe ID

When your AdobeID has been whitelisted, you will need to create a device token using your AdobeID and API key. To do this, visit: https://aex.publish.adobe.com/index.html and enter your API key ( also referred to as your IMS Client ID ). Click the “IMS Authorize” button. You will be asked to log in with an AdobeID, use the one that was whitelisted above.

After logging in you will be given a Device Token, Device ID, and expiration time. A note about the Device Token + Device ID:

  • The device token will expire after 6 months
  • The device token will automatically be refreshed, as long as it’s being actively used. So it is not necessary to generate a new device token after 6 months.
  • A password update will invalidate the device_token in the future (within a 2 to 3 month timeframe)

___

4. Entering the API information into the plugin

After obtaining the: Adobe API Key, Adobe API Secret, Device Token, and Device ID you can then enter these pieces into the Digital Publishing plugin for WordPress. You can do this by clicking on Settings on the left hand side, then the Adobe API tab. Make sure your API version is set to Publish (v2.0). After entering the following fields:

  • Adobe API Key
  • Adobe API Secret
  • Device Token
  • Device ID

Click Save. The plugin will then automatically create an access token for you behind the scenes. You should see it populated in the Adobe API tab after the save. You should also see a list of authorized publications at the bottom of the tab that will show you the publications your AdobeID has access to. If you need access to a publication that does not appear in this list, log into the Adobe Digital Publishing Solution portal and assign the AdobeID (the one assigned to the API) access to the publication you’d like to use.

Working with the Plugin


Installation

You can also download a copy of the plugin above then drag the “digital-publishig-for-adobe-publish” folder into your WordPress plugins folder.

After you activate the plugin you will see a menu item on the left for “Digital Publishing.” Clicking on this will launch you into the Digital Publishing plugin.


Recommended Hosting Services

We’ve been using Media Temple’s Wordpress Premium hosting service with our clients and the plugin works great with the service.


Setting up the plugin: Settings

Opening the plugin for the first time will take you into the settings area. Here you can set-up the plugin and assign any defaults.


General

You can use the plugin in two modes:

Normal

The plugin will integrate into WordPress’ native views. You will see submenus in the Digital Publish navigation item on the left.

Full App

There will be one icon for Digital Publish on the left navigation. Clicking this icon will launch you into a full screen view of the app with our custom UI / views

Adobe API

If you have access to the Adobe API (see how to get access to the Adobe Digital Publishing Solution API) you can enter them here. If you choose not to enter an Adobe API you can still use the plugin. Cloud functions will be disabled.


Target Devices

COMING SOON: The ability to have renditions for different devices. You can set the different device sizes in this tab


Templates

This tab will show you the templates the plugin has found. You can use this tab to make sure the templates are available and even set the default template used when creating new articles. [see working with templates for more information about templates]


Advanced

As additional features come up we’ll be adding their options to the advanced tab.

Article List View

Working with Articles

Articles are an integral part of the plugin. The plugin allows you to easily migrate existing content or create new content as Adobe Digital Publishing Solution articles. We make working with, editing, and publishing articles easy. Clicking on the “Articles” section in the plugin will take you to the articles list.


Creating a new article

Articles can be created from scratch. Clicking the (+) button will allow you to create a new article. When creating a new article, the name must be unique. Enter an article name and click create. The new article will appear at the top of the article list.


Importing Articles

You can also work with existing content already created in WordPress. To import an article, go to your list of posts (or custom taxonomy posts). In the standard WordPress list, check the posts you’d like to import as articles. From the bulk actions drop down menu, select “Import as DPS Article” and click “Apply.” This will begin the process of importing the selected content into articles.

When importing articles, the plugin keeps everything associated with the original article: custom fields, taxonomies, categories, tags, etc. A reference to the original post is stored as well, so you can sync any changes made from the original post (if it changes).

In addition, the plugin also takes any taxonomy or category values and adds them as “internal keywords” to make it easier to search and manage articles in the Adobe Digital Publishing Solution web portal.

After the import, if you go back to the articles list in the plugin, you will see the new article(s).



Editing Articles

You can make changes to an article by either editing the original post in wordpress and then syncing the changes or by clicking on an article inside of the Digital Publishing Tools for Wordpress. When you open an article in the plugin you’ll see a preview of how it looks with the selected template.

  • The properties drop down allows you to edit any associated Adobe Digital Publishing Solution metadata.
  • The Edit button allows you to edit the article using wordpress. NOTE: editing an article in the plugin does not sync the changes back to the original post. If the intent is to edit the source file you’ll want to edit the original post and then sync the changes back to the article in the plugin.


Deleting Articles

Articles can be deleted one by one or in bulk. Opening an article in the plugin will give you the ability to delete it. Clicking the trash can icon will ask you if you’d like to delete the article. If the article is also hosted in the cloud, you will also be given the option to delete it from the cloud at the same time. Once an article is deleted it can not be undone. Similarly, if you’re in the article list, you can select a group of articles and click the trash can icon to delete them.


Downloading Articles

If you would like a copy of the article for backup or to upload manually into the Adobe Digital Publishing Solution cloud, you can download a copy of the article. When you open an article in the plugin, there’s an download icon. Clicking the icon will download a zip of the following:

  • Any content assets (thumbnail or social share image)
  • A sidecar XML file of the article’s metadata
  • An .article file (a zip of the HTML article and it’s assets) ready for upload into Adobe Digital Publishing Solution




Working with the Articles and the Adobe Digital Publishing Solution cloud.

When creating and working with articles in WordPress, they will be local to that install of WordPress. When you want to publish them you can use the cloud functions. Any changes made to articles in Wordpress are made locally. You will need to update the changes you’ve made to see them reflected in the Adobe Digital Publishing Solution cloud.

When you open an article, there’s a cloud button with a drop down of options.

Create in the Cloud

When you first create an article, it’s only local to that instance of WordPress. The only option you’ll have when clicking on the cloud button is to “Create in the Cloud.” This will create a new article in the Adobe Digital Publishing Solution cloud with all of the metadata of the article.


Update Metadata

This will only update the article’s metadata.


Update Content (thumbnails)

This will only upload updates to the article’s thumbnail or social sharing images


Update Article Folio

This will only update the article content file (the .article file) that’s attached to the article


Update ALL

Update all will update the article’s metadata, contents, and article folio (.article).


Delete from the Cloud

This will delete the article from the Adobe Digital Publishing Solution cloud


Unlink from Cloud

This will unlink the local WordPress article with the one in the cloud. You can re-link the article at a later time if you do this.


Link to Cloud

This will bring up a list of article already in the cloud. You can link this local WordPress article with the one in the cloud.


Publish

This will publish the article in the Adobe Digital Publishing Solution cloud.
